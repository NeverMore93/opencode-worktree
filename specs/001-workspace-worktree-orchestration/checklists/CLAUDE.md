# CLAUDE.md

## Responsibility Boundary
- This directory owns requirement-quality checklists for the `001-workspace-worktree-orchestration` feature.
- These files test the quality of the written requirements; they do not test runtime behavior or implementation correctness.

## Allowed Content
- Checklist markdown files such as `requirements.md`
- Follow-up checklist notes that identify gaps, ambiguities, and conflicts in the feature spec
- Additional checklist files scoped to a distinct review angle for the same feature

## Forbidden Content
- Test plans for executing code
- Implementation notes, task execution logs, or design decisions that belong in `spec.md` or `tasks.md`
- Checklists for unrelated features
