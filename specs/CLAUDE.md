# CLAUDE.md

## Responsibility Boundary
- This directory owns feature-specific design artifacts produced by the SpecKit workflow.
- Each child feature directory should stay self-contained and should not be used as a general notes area.

## Allowed Content
- Feature directories named by feature branch or identifier
- Specs, plans, tasks, quickstarts, contracts, checklists, context, and case notes scoped to one feature
- Small cross-feature guidance only when it applies to the whole `specs/` tree

## Forbidden Content
- Product source code or automation scripts
- Top-level repository docs unrelated to feature design
- Mixed feature content stored without a dedicated feature directory
